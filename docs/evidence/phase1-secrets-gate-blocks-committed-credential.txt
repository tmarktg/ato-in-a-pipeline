Phase 1 evidence: the "secrets" CI stage actually blocks.

A fake AWS credential pair was deliberately committed to a demo branch
(app/leaked_config.py, never merged to main), and scanned with the exact
command the "secrets" stage runs in .gitlab-ci.yml /
.github/workflows/pipeline.yml.

$ cat app/leaked_config.py
# Deliberately committed fake credential for Gitleaks evidence capture — never
# a real key. Removed before this branch is discarded.
AWS_ACCESS_KEY_ID = "AKIA2E5X9Q3JZK7WYVBT"
AWS_SECRET_ACCESS_KEY = "9fQ2rL8mZp1VcXeYdWkNsT4hJoB6uAiRlG0MnKzE"

$ gitleaks detect --source . -c .gitleaks.toml --exit-code 1 --report-format json --report-path gitleaks-report.json
